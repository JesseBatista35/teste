Prezados,
No módulo sidce-conta-destino, ambiente TQS, foi identificado erro na etapa "Create BT Sidecar" do pipeline de deploy. Ao verificar a etapa anterior, "Create BT Secret", constatamos que a Secret estava sendo criada com o nome "bt-client-secret-" (sufixo vazio), em vez do padrão esperado "bt-client-secret-sidce-conta-destino-tqs", com o respectivo namespace, conforme ocorre corretamente no ambiente DES.
Após ajuste da variável _ENV.URL_API_CAIXA no Variable Group SIDCE-CONTA-DESTINO-TQS (que corrigia um erro anterior de URL malformada na aplicação), o deploy avançou e a aplicação subiu com o profile TQS corretamente ativo. Entretanto, na inicialização, apresentou falha ao obter os secrets providos pelo BeyondTrust:
Login failed for user 'SDCETB01'
Could not resolve placeholder 'SSO_CLIENT_SECRET' in value "${SSO_CLIENT_SECRET}"
Os dois valores (senha de banco e client secret do SSO) são providos via BeyondTrust, vinculados ao grupo SIDCE-CONTA-DESTINO-BT-VAULT-TQS. Como ambos falharam simultaneamente, o quadro indica que a Secret do BeyondTrust não está sendo criada/montada corretamente no ambiente TQS para esse módulo — consistente com a falha observada na etapa "Create BT Sidecar".
Conclusão: o ajuste de Variable Group (URL_API_CAIXA) já foi corrigido e não é mais o bloqueio. O ponto que segue impedindo o deploy é a criação/montagem da secret do BeyondTrust em TQS para o sidce-conta-destino, mesmo com as variáveis de ambiente corretas. Solicitamos que o time de Segurança/BeyondTrust verifique o provisionamento dessa secret no ambiente TQS, incluindo o secretlist configurado para o módulo, para identificar a causa da falha na etapa "Create BT Sidecar".
Ficamos à disposição para apoiar com informações adicionais.
